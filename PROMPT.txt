想做一个在线数独游戏，用 React 做。规则大家都懂，9x9 网格分 9 个 3x3 宫，每行每列每宫包含 1-9 不重复。游戏流程大概是：系统生成一道题目→展示给用户→用户填数字→可以暂停/继续→可以提示→可以撤销→完成时庆祝。

难度分级有四个：入门（45 个空格）、简单（50 个空格）、中等（55 个空格）、困难（58 个空格）。空格越少越难。生成数独的时候要保证有唯一解，不能生成一个多解的题目。生成算法大致是先填满一个完整的数独然后挖洞，每挖一个洞要验证剩下的空格是否仍然有唯一解（用求解器跑一遍），如果不是唯一解就把这个洞填回去换个位置挖。挖洞验证是个耗时操作，可以用候选数标记 + 简单回溯来加速。

状态机：游戏状态包括 idle（未开始）、generating（生成中）、playing（进行中）、paused（暂停）、solved（已完成）、failed（无解或冲突）。idle 用户点开始→generating 生成题目→playing 用户填写。playing 过程中可以点暂停→paused 再次点开始→playing。playing 过程中所有格子填完且无冲突→solved 弹胜利动画。playing 过程中某行/列/宫出现重复数字→进入 failed 模式高亮冲突格子但不强制清空（用户自己改）。

撤销功能：用户每填一个数字记录到历史栈里，点撤销弹栈把那个格子恢复成空。提示功能：选一个空格，系统帮他填一个候选数（或者把那个格子高亮一下），提示次数用完了就不能再提示。提示不要直接告诉答案，应该是"这个格子应该是 X"还是"这个格子只剩一个候选数了"这种引导式的。

求解器：游戏内置一个求解器，用户填错或者卡住的时候可以一键调用求解器把当前进度解出来（前提是当前局面有解）。求解器要支持"获取一个候选数"（不破坏当前局面）和"完全求解"（直接解完整个棋盘）两种模式。求解器基于回溯+候选数剪枝，跑得很快毫秒级。

时间统计：playing 状态开始计时，solved 的时候显示总共用了多长时间、提示用了多少次、撤销了多少次。paused 状态暂停计时。

UI 方面：9x9 网格用 div+CSS Grid 实现，3x3 宫的边界用粗一点的黑线区分。数字按钮 1-9 在底部或者右侧，点数字按钮再点格子就填进去，或者直接键盘按 1-9 填到当前选中的格子。已填的初始数字（题目给的）用粗体黑色，用户填的用普通字体蓝色，冲突的高亮红色背景。候选数小字（如果做了候选数标记功能）显示在格子角落小九宫格。

历史回放：用户玩完之后想"刚才我是在哪一步开始走错的"，可以回放整个填写过程。系统记录每一步的格子位置和填入数字，回放的时候按 0.5 秒一步自动播放，用户也可以手动下一步/上一步。

本地存储：用户的游戏进度（当前局面、状态、计时、撤销栈）存到 localStorage，刷新页面不丢。如果同时开多个 tab 用 BroadcastChannel 同步一下状态避免一个 tab 玩另一个 tab 不知道。题目生成有随机种子，用户想分享"我做出来这道题用了 X 分钟"的时候 URL 带个 seed 参数，接收方打开同一个 seed 复现同一道题。

坑点提示：挖洞时唯一性验证要优化不然 4 个难度都跑一遍用户要等几十秒。撤销栈深的时候占用内存大要考虑截断或者只记录差量。求解器遇到多解局面要能识别出来不能假装"解出来了"实际给的是第一个解。

附加功能再来几个。错误高亮：用户填了冲突数字时不仅格子变红，关联的同区域格子（同宫/同行/同列）如果有相同数字也高亮提示。自动笔记：用户每填一个数字自动帮他在笔记区记录候选数（这个格子还能填什么），笔记区小数字在格子角落。检查功能：用户填完不确定自己有没有错，点检查高亮所有冲突格子但不清空让用户自己改。

分享这道题的功能入口是另一个隐含功能。生成题目后点"分享"按钮，把当前的局面（已填的初始数字）和 seed 编码到一个短 URL（比如 sudoku.app/p/abc123），好友打开同一个 URL 复现同一道题。这跟回放功能用的不是同一个 URL——回放是看历史，分享是做题。

难度评级要客观。不能在生成时按"挖洞数"硬定难度，应该是用求解器跑一遍看用了多少高级技巧（naked single、hidden single、naked pair 之类），用到的技巧越高级难度越高。简单题只需要 naked single 就能解出来，中等题需要 hidden single，难的需要更复杂的链式推理。生成后跑一遍评估给一个客观评级。

输入方式再增强。除了点击格子和数字按钮，支持键盘：方向键移动光标、数字键填数、Backspace 清除当前格、Esc 退出当前选中。触屏设备上长按格子弹数字小键盘选数。Pencil mode（笔记模式）开关：开启后输入数字不直接填到格子里而是作为候选数小字标记。

完成庆祝的动画。solved 状态下背景渐变色变化从灰到彩色，9x9 网格格子依次闪烁一遍，所有格子高亮一下然后回归正常色，整个过程 1.5 秒。完成后弹一个 toast"太棒了！你用了 X 分 Y 秒完成！"带个关掉的按钮。
